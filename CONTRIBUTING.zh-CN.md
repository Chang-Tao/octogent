# 贡献指南

Octogent 是一个实验性的个人项目，目前未积极审查拉取请求。如果你仍然决定提交，请保持更改范围小、有测试支持且易于审查。

## 在修改任何内容之前

- 阅读 `docs/` 中的相关文档
- 在增加更多功能之前，检查行为是否已存在于 API 或 UI 中
- 在文档和产品框架中保持项目以 Claude Code 为主
- 不要将推测性功能当作已实现的功能来记录

## 先决条件

- Node.js `22+`
- pnpm
- `claude`（用于支持的代理工作流）
- `git`（用于工作树功能）

## 设置

```bash
pnpm install
```

## 开发

```bash
pnpm dev
```

开发运行器会同时启动本地 API 和 Web 应用。

## 必需的检查

在提交拉取请求之前运行以下命令：

```bash
pnpm test
pnpm lint
pnpm build
```

如果需要重新格式化代码，请使用 `pnpm format`。

## 优质贡献的特征

- 范围清晰的增量更改
- 为行为更改编写测试
- 当工作流或概念发生变化时，在同一更改中更新文档
- 代码和文档反映当前实现，而非路线图

## 文档政策

- `docs/` 用于贡献者和未来代理的理解
- 如果你更改了触手、待办事项、终端、编排或消息传递，请更新相应的文档页面

## 拉取请求预期

- 请理解目前拉取请求不会得到积极审查
- 用一段简短文字解释问题
- 用具体术语解释行为变化
- 说明任何持久化、API 或工作流方面的影响
- 对于可见的 UI 更改，请附上截图
- 如果代码是使用 AI 编写的，请说明使用的编码代理和模型
- 明确提及缺失的后续工作，而不是将其隐藏

## 当前最重要的领域

- 触手模型和面向代理的上下文文件
- 待办事项解析和委派流程
- Claude Code 终端生命周期
- 子代理编排
- 代理间消息传递
- 修复现有问题并优化可靠性
